# 사용자 흐름 (User Flow)

> 키워드 기반 카드뉴스 자동 생성 웹앱의 사용자 여정과 페이지 간 이동 흐름을 정의한다.

## 1. 페르소나 & 핵심 시나리오

### Primary Persona — "콘텐츠 마케터 / SNS 운영자"
- **상황**: 매일 SNS에 게시할 카드뉴스 콘텐츠가 필요
- **목표**: 키워드 하나만 입력해 빠르게 카드뉴스 생성, 즉시 다운로드 또는 공유
- **제약**: 디자인 도구 사용 미숙, 정보 검색 시간 부족

### 핵심 시나리오
1. 첫 방문 → API 키 등록 → 키워드 입력 → 카드뉴스 생성
2. 재방문 → 키워드 입력 → 템플릿 변경 → 카드뉴스 생성 → 결과 저장

---

## 2. 정보 구조 (IA)

```
홈 (/)
├─ 키워드 입력
├─ 템플릿 선택 (간단)
└─ [시작] → /process

/process (전체 파이프라인 진행 현황)
├─ Step 1 키워드 입력 → /process/1
├─ Step 2 웹 검색      → /process/2
├─ Step 3 정보 정리    → /process/3
├─ Step 4 템플릿 구성  → /process/4
├─ Step 5 이미지 배치  → /process/5
└─ Step 6 카드뉴스 완성 → /result

/process/[step] (단계별 상세)
└─ 각 단계의 입력/출력/로그 확인

/result (완성 카드뉴스)
├─ 카드 슬라이드 미리보기
├─ 다운로드 (PNG/PDF/ZIP)
└─ 히스토리 저장

/templates (템플릿 관리)
├─ 기본 템플릿 목록
├─ 신규 템플릿 추가
└─ 편집/삭제

/settings (API 키 관리)
├─ LLM 설정
├─ Web Search 설정
├─ Image Search 설정
└─ AI Image Gen 설정 (선택)
```

## 3. 메인 사용자 흐름 (Happy Path)

```
┌──────────────────┐
│  첫 방문         │
│  (/)             │
└────────┬─────────┘
         │ API 키 미등록 감지
         ▼
┌──────────────────┐
│  배너 노출       │
│ "API 키를 먼저   │
│  등록해주세요"   │
└────────┬─────────┘
         │ [설정으로 이동]
         ▼
┌──────────────────┐
│  /settings       │
│  API 키 입력     │
│  연결 테스트     │
└────────┬─────────┘
         │ 저장 성공
         ▼
┌──────────────────┐
│  /               │
│ 키워드 입력 폼   │
│ 템플릿 선택      │
└────────┬─────────┘
         │ [카드뉴스 생성 시작]
         ▼
┌──────────────────┐
│  /process        │
│ 6단계 타임라인   │
│ 실시간 상태 표시 │
└────────┬─────────┘
         │ Step 1~5 자동 진행
         │ (각 단계 클릭 → 상세 보기)
         ▼
┌──────────────────┐
│  /process/[N]    │
│ 단계 상세        │
│ 입력/출력/로그   │
└────────┬─────────┘
         │ 모든 단계 완료
         ▼
┌──────────────────┐
│  /result         │
│ 카드뉴스 슬라이드│
│ 다운로드/공유    │
└──────────────────┘
```

## 4. 분기 흐름 (Edge Cases)

### 4.1 API 키 미등록 상태에서 생성 시도
```
/ → [시작 버튼 클릭]
  → 모달 "API 키 미등록"
  → [지금 등록] → /settings
  → [취소] → / (홈 유지)
```

### 4.2 단계 진행 중 오류 발생
```
/process (Step N 오류)
  → 해당 단계 카드 빨간색 표시
  → [재시도] 또는 [건너뛰기] 옵션 제공
  → 재시도 실패 시 → [처음부터 다시 시작] 또는 [부분 결과 보기]
```

### 4.3 이미지 검색 실패 → AI 생성 폴백
```
Step 5 (이미지 배치)
  → 웹 이미지 검색
  → 결과 부족 / 실패
  → [자동 폴백] AI 이미지 생성 (IMAGE_GEN_API_KEY 존재 시)
  → 키 미등록 시 → 플레이스홀더 이미지 사용 + 경고 표시
```

### 4.4 템플릿 변경 (생성 도중)
```
/process (Step 4 완료 후)
  → 사용자가 [템플릿 변경] 클릭
  → 모달에서 다른 템플릿 선택
  → Step 4부터 재실행
```

### 4.5 히스토리 진입 (재방문)
```
/ → 우측 상단 [히스토리]
  → 드로어 또는 /result?id=xxx
  → 이전 생성 카드뉴스 확인
  → [재생성] 또는 [삭제]
```

## 5. 페이지 간 이동 매트릭스

| From | To | 트리거 |
|------|-----|--------|
| `/` | `/settings` | "API 키 등록" 배너/버튼 |
| `/` | `/templates` | "템플릿 관리" 메뉴 |
| `/` | `/process` | "카드뉴스 생성 시작" 버튼 |
| `/process` | `/process/[step]` | 단계 카드 클릭 |
| `/process/[step]` | `/process` | 뒤로가기 / 닫기 |
| `/process` | `/result` | Step 6 완료 자동 이동 |
| `/result` | `/` | "새로 만들기" 버튼 |
| `/result` | `/process` | "다시 생성" 버튼 |
| 모든 페이지 | `/settings` | 우측 상단 톱니바퀴 |
| 모든 페이지 | `/` | 좌측 상단 로고 |

## 6. 상태 전이 (State Diagram)

### 파이프라인 단계별 상태
```
[idle] ──start──▶ [running] ──success──▶ [completed]
                      │
                      └──error──▶ [failed] ──retry──▶ [running]
                                       │
                                       └──skip──▶ [skipped]
```

| 상태 | UI 표시 | 사용자 액션 |
|------|---------|-------------|
| idle | 회색 점선 카드 | 없음 (대기) |
| running | 파란색 + 펄스 애니메이션 | 단계 클릭 시 실시간 로그 |
| completed | 녹색 체크 + 결과 미리보기 | 클릭 → 상세 보기 |
| failed | 빨간색 + 에러 메시지 | [재시도] / [건너뛰기] |
| skipped | 회색 + "건너뜀" 라벨 | 클릭 → 사유 보기 |

## 7. 반응형 분기

| Breakpoint | 흐름 차이 |
|-----------|-----------|
| 320 (mobile) | `/process` 단계 카드 세로 스택, 결과 미리보기 1열 |
| 768 (tablet) | 단계 카드 2x3 그리드, 결과 2열 |
| 1024+ (desktop) | 단계 타임라인 가로 진행 바, 결과 3~4열 |

---

## 8. 접근성 흐름

- 모든 페이지: 키보드만으로 전체 흐름 완주 가능 (Tab → Enter)
- 스크린 리더: 각 단계 상태 변경 시 `aria-live="polite"`로 알림
- 진행 중 자동 페이지 이동은 사용자 동의 후에만 (옵션 토글 제공)
- 색상만으로 상태 구분 금지 → 아이콘 + 텍스트 라벨 병행
